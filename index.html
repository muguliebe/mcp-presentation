🤔MCP?

Model Context Protocol

AI와 외부 시스템을 연결하는 표준 프로토콜

쉽게 말하면

USB-C 가 다양한 주변기기 및 액세서리에 연결하는 표준화된 방식을 제공하는 것처럼

MCP는 AI 모델을 다양한 데이터 소스 및 도구에 연결하는 표준화된 방식을 제공합니다.

🧠AI = 두뇌 + 입

(생각하고 말하기만 가능)

👁️MCP = 눈 + 귀 + 손

(보고, 듣고, 행동하는 능력 부여)

10분 -> 1분

Before

  • 에러 발생 → 개발자가 직접 로그 확인
  • 내가 모르는 소스 코드 분석
  • 다수 인원이 중복 확인
  • Git 변경사항 수동 추적
  • 원인 파악까지 긴 시간 소요

⏱️소요시간: 5분~30분

After (with MCP)

  • 에러 발생 → AI가 자동 분석
  • 워크스페이스 소스 코드 접근
  • Git 변경사항 자동 분석
  • 프로젝트 컨텍스트 이해
  • 근본적인 원인 파악 및 해결책 제안

⚡소요시간: 30초~2분

🔧MCP 기능 3가지 활용

🛠️

Tools (도구)

AI가 할 수 있는 작업

  • 에러 모니터링 시작
  • AI 분석 실행
  • 알림 전송
  • 데이터 수집
🎯

Sampling (질의)

AI에게 질문

  • 에러 원인 분석 요청
  • 해결책 생성 요청
  • 컨텍스트 기반 추론
  • 전문가 수준 분석
📚

Resources (자원)

AI가 읽을 수 있는 데이터

  • 워크스페이스 소스 코드
  • Git 변경 이력
  • 설정 파일
  • 로그 파일

핵심 포인트

AI가 "에러 모니터링"과 "분석 실행"이라는 도구를 사용할 수 있게 정의

🔗2단계: 각 도구별 실제 함수 연결

// 각 도구에 대응하는 실제 함수 매핑
this.server.setRequestHandler(CallToolRequestSchema, async (request) => {
    const { name, arguments: args } = request.params;
    
    switch (name) {
        case 'start_error_monitoring':
            return await this.startErrorMonitoring(args);
        case 'analyze_with_ai':
            return await this.analyzeWithAI(args);
        default:
            throw new Error(`Unknown tool: ${name}`);
    }
});

핵심 포인트

AI가 "analyze_with_ai"를 호출하면 실제 분석 함수가 실행됨

🤖3단계: AI에게 분석 요청

// MCP Sampling으로 AI 분석 요청
const samplingRequest = {
    method: 'sampling/createMessage',
    params: {
        messages: [{
            role: 'user',
            content: {
                type: 'text',
                text: `에러 분석 요청: ${errorMessage}`
            }
        }],
        systemPrompt: "당신은 Spring Boot 에러 분석 전문가입니다...",
        modelPreferences: {
            hints: [{ name: 'claude', priority: 1.0 }]
        }
    }
};

const response = await this.server.request(samplingRequest);
